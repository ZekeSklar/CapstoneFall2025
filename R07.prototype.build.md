## Prototype Build — Current Implementation

### Implemented Features
- Admin → Printers: Live Device Status panel with manual Refresh and auto‑refresh on load.
- Admin index: “Printer Device Alerts” (attention/SNMP faults) + low inventory section.
- Manager dashboard: per‑printer status grid with Refresh One/All; auto‑refresh every 5 minutes.
- Ticketing: supply/issue tickets with email notifications; group scope option.
- Issue rate limiting: max 3 issue reports per printer per hour; users still receive “thank you.”
- Daily issue summary email: interval‑based send to `.env` recipient(s) and subscribed superusers.

### How to Run (dev)
1) From `codebase/`, create `.venv` and install requirements (Django, python‑dotenv, pysnmp, import‑export).
2) Configure `codebase/.env`:
   - Mail: `EMAIL_BACKEND`, `EMAIL_HOST`, `EMAIL_PORT`, `EMAIL_USE_TLS`, `EMAIL_HOST_USER`, `EMAIL_HOST_PASSWORD`, `DEFAULT_FROM_EMAIL`, `EMAIL_TO`
   - SNMP: `SNMP_COMMUNITY` (e.g., `public`), `SNMP_TIMEOUT=5`, `SNMP_RETRIES=1`, `SNMP_POLL_INTERVAL_SECONDS=300`
3) `python manage.py migrate`
4) `python manage.py runserver`

### Verifying Live Status
- Set a printer’s `ip_address` and ensure SNMP is enabled on the device.
- Open Admin → Printers → select the device; observe the Live Device Status; click Refresh.
- Manager dashboard shows the same data for assigned groups.

### Notes
- Some models expose limited Printer‑MIB data; alerts/supplies may be empty even when status/error flags work.
- SNMP uses v2c with fallback to v1; timeouts kept low to preserve UX.
